# 00 — 전체 사이클 개요

## 풀려는 문제

외국인 투자자가 한국 상장주식을 사려면 지금은 세 가지 길뿐이고, 셋 다 마찰이 크거나 남의 협조에 의존한다.

| 방법 | 내용 | 한계 |
|------|------|------|
| 직접투자 | 국내 증권사 계좌 개설 후 직접 매매 | 계좌 개설·상임대리인·서류 공증·환전. 절차가 무겁다 |
| 외국인 통합계좌 | 해외 증권사 명의 계좌로 일괄 주문·결제 | 내가 쓰는 증권사가 서비스를 열어야 하고, 명의는 브로커 앞 |
| ADR·예탁증서 | 해외 거래소에서 달러로 매매 | **발행회사가 상장한 종목에만 존재** |

→ 임의의 상장 종목에 투자자가 스스로 접근하는 경로가 없다. 이 레포는 **네 번째 길**을 만든다.

---

## 네 개 트랙

```
① 규제          진짜 주식을 온체인에 올릴 수 있는가
   ↓ (청구권이 아닌 전자등록주식으로 만들 근거를 확보)
② 인프라        자산과 대금을 안전하게 주고받을 수 있는가
   ↓ (두 원장 + DvP + 외부 데이터 공급)
③ 토큰화        발행·보유·배당·매매·환매가 도는가
   ↓ (규제를 코드로 강제, 대기는 1차에만)
④ 에이전트      사람이 실제로 쓸 수 있는가
                (자연어 주문, 규칙은 그대로)
```

각 트랙은 앞 트랙이 남긴 문제를 푼다. 하나라도 빠지면 뒤가 성립하지 않는다.

## 핵심 설계 결정 4가지

### D1. 토큰 = 전자등록주식 자체 (청구권 아님)
발행회사가 **발행인계좌관리기관**이 되어 투자자별 계좌부를 분산원장으로 직접 관리한다.
그 원장이 법적 효력을 갖는 전자등록계좌부이므로, 온체인 이전이 곧 주식 양도다.
→ 상세: [01-regulation.md](./01-regulation.md)
> ⚠️ 실패 시: 발행인계좌관리기관 등록 없이 배포하면 토큰이 법적 효력 없는 청구권이 된다.

### D2. 규제 판단은 오프체인, 집행은 온체인
개인정보와 재량적 판단은 오프체인에 남고, 온체인에는 검증 결과의 서명값만 올라가 결정론적으로 집행된다.
→ 상세: [01-regulation.md](./01-regulation.md)
> ⚠️ 실패 시: 판단 로직을 온체인에 넣으면 개인정보보호법 위반 위험, 법령 개정 시 컨트랙트 전체 재배포 필요.

### D3. 미리 수탁 → 2차 매매는 즉시, 1차만 대기
대상 주식이 미리 수탁돼 있으므로 투자자 간 매매는 원장에서 소유자만 바꾸면 끝난다.
T+2·환전 시차는 수탁 물량 자체가 바뀌는 발행·환매에만 발생한다.
→ 상세: [03-tokenization.md](./03-tokenization.md)
> ⚠️ 실패 시: 2차 매매에 대기를 넣으면 실물 뒷받침 없이 대기 상태 토큰이 유통되어 INV-7 위반.

### D4. 자산 원장과 결제 원장 분리
자산은 퍼블릭 유동성이, 대금은 즉시 완결성이 필요하다. 요구가 다르므로 원장을 나누고 DvP로 잇는다.
→ 상세: [02-infrastructure.md](./02-infrastructure.md)
> ⚠️ 실패 시: 단일 원장에 넣으면 결제 확정 전 자산이 이전되거나, 법적 완결성 7일 내 되돌림이 발생할 수 있다.

## 액터

| 액터 | 역할 | 온체인/오프체인 |
|------|------|------------------|
| Investor | 주식 토큰 보유·매매 주체 (Alice, Bob) | 온체인 지갑 + 오프체인 신원 |
| IssuerAccountManager | 발행인계좌관리기관. 투자자별 계좌부(=온체인 원장) 운영 | 온체인 원장 운영 |
| Custodian | 실물 주식 수탁, 1차 발행·환매 시 실물 매매 | 오프체인 (어댑터로 연결) |
| ClaimIssuer | KYC·적격성·관할 등 검증 후 클레임 발급 | 오프체인 심사 → 온체인 서명 |
| Oracle | 외국인 한도·가격·기업행위 데이터 공급 | 오프체인 원천 → 온체인 전달 |
| Orchestrator | 1차 시장 DvP의 순서·commit/abort 조율 | 오프체인 서비스 |
| Agent | 자연어 주문을 온체인 실행으로 변환 | 오프체인 서비스 |

## 전체 흐름 (한 바퀴)

```
[온보딩]     Investor → ClaimIssuer 심사 → ONCHAINID에 클레임 등재
                                              │
[2차 매수]   Investor ──매수──> 주식 토큰 원장 (신원·한도 검사 → 즉시 확정)
                                              │
[배당]       Custodian 배당 수령 → 투자자별 조약 세율 적용 → 분배
                                              │
[2차 매도]   Investor ──매도──> 다른 Investor  (수신자 검사 → 즉시 확정)
                                              │
[1차 발행]   수탁 물량 증가 필요 시:
             대금 에스크로 → 실물 매수(T+2·환전) → 검사 → 발행 → 완료
[1차 환매]   수탁 물량 감소 필요 시:
             토큰 잠금 → 실물 매도(T+2·환전) → 소각 → 대금 반환
                                              │
[에이전트]   위 모든 동작을 자연어로 지시 (규칙은 동일하게 적용)
```

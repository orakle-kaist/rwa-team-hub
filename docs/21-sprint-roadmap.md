## 라이브 데모 목표 시나리오

### 필수 (Week 6까지)

| # | 시나리오 | 완료 기준 |
|---|----------|-----------|
| 1 | Alice 온보딩 | KYC·적격성·관할 클레임 발급 상태 온체인 확인 |
| 2 | "SK하이닉스 100주 사줘" | 5개 규제 검사 통과 → 2차 시장 즉시 체결 → 잔고 변화 |
| 3 | "Bob에게 40주 팔아줘" | 수신자 신원 검사 → 즉시 이전 → 양쪽 잔고 확인 |
| 4 | 거절 시나리오 | "한도 초과 종목 사줘" → 에이전트 한국어 설명 |
| 5 | 배당 | Alice(미국 15%) · Bob(독일 20%) 차등 세율 분배 결과 |

### 가능하면 (Week 5 여유 시)

| # | 시나리오 |
|---|----------|
| 6 | 1차 발행 흐름: requestDeposit → T+2 시뮬레이션 → Claimable → claim |

### PoC 단순화 합의 사항

| 항목 | 단순화 내용 |
|------|------------|
| 2차 시장 | `IStockToken.transfer()` 직접 호출 (오더북·DEX 없음) |
| 외국인 한도 예상비율 | `currentRatio + Δ` 단순 산식 고정 (D-B7) |
| 조세 거주성 | 국가 코드 클레임 2종 (미국 15%, 기타 20%) 허용 (D-B8) |
| 멱등성 | requestId 중복 체크만 (D-B9) |
| 듀얼 원장 | Anvil 단일 인스턴스 대체 (Besu 연동 skip) |

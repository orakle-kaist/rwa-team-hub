# 12 — 인수 시나리오

이 문서의 시나리오가 모두 통과하면 구현 완료다.
**테스트 이름에 `AC-x.y` 식별자를 포함한다.** 예: `test_AC_2_1_secondaryBuySettlesInSameTx()`

기준 시나리오 인물: **Alice**(두바이 거주 외국인), **Bob**(다른 외국인 투자자), 종목 **SK하이닉스**.

---

## AC-1. 신원 · 온보딩

| ID | 시나리오 | 기대 결과 |
|----|----------|-----------|
| AC-1.1 | 필수 클레임을 모두 보유한 Alice의 `isVerified` 확인 | `true` |
| AC-1.2 | KYC 클레임이 없는 주소로 이전 시도 | revert, 모듈 = IdentityEligibility |
| AC-1.3 | 클레임이 철회된(`removeClaim`) 주소로 이전 시도 | revert |
| AC-1.4 | 온체인 저장소에 개인정보/해시가 존재하지 않음 | 정적 검사 통과 (INV-4) |

---

## AC-2. 2차 시장 — 즉시 체결 (핵심)

| ID | 시나리오 | 기대 결과 |
|----|----------|-----------|
| **AC-2.1** | Alice가 미리 수탁된 토큰 100주를 매수 | **단일 트랜잭션에서 완결.** 대기 상태 없음 (INV-6) |
| AC-2.2 | 매수 시 5개 컴플라이언스 모듈이 모두 호출됨 | 호출 로그 확인 (INV-3) |
| AC-2.3 | Alice가 Bob에게 40주 매도 | 즉시 확정, Alice 60 / Bob 40 |
| AC-2.4 | 클레임 없는 Bob에게 매도 시도 | revert, 잔고 변화 없음 |
| AC-2.5 | 2차 경로에 request/claim 패턴이 존재하지 않음 | 코드 검사 (R4) |
| AC-2.6 | 이전 성립 시 계좌부 총량과 잔고 합이 일치 | INV-1 |

## AC-3. 외국인 보유한도

| ID | 시나리오 | 기대 결과 |
|----|----------|-----------|
| AC-3.1 | 한도 여유 상태(32.0% → 32.4%)에서 매수 | 통과 |
| AC-3.2 | 이전 후 비율이 법정 한도(예: 49%)를 초과 | revert, `ForeignLimitExceeded` |
| AC-3.3 | revert에 종목코드·법정한도·예상비율·오라클 시각이 포함됨 | 필드 검증 |
| AC-3.4 | 오라클 데이터가 `maxAge` 초과(스테일) | **보수적 거절** (INV-9) |
| AC-3.5 | 컨트랙트가 자체 잔고 합산으로 비율을 계산하지 않음 | 코드 검사 (INV-5) |
| AC-3.6 | 한도 초과 상태에서 **매도**는 허용 | 빠져나가는 방향은 차단하지 않음 |

## AC-4. 발행(mint) 경로 컴플라이언스

| ID | 시나리오 | 기대 결과 |
|----|----------|-----------|
| **AC-4.1** | 화이트리스트 미등재 주소로 mint 시도 | **revert** (기본 ERC-3643이면 통과해버림 — R6 대응 확인) |
| AC-4.2 | 외국인 한도 초과 상태에서 mint 시도 | revert |
| AC-4.3 | 락업 대상 수량에 대한 mint | 정책대로 차단 또는 락업 표시 |
| AC-4.4 | 정상 조건에서 mint | 성공, `totalSupply` 증가 |

---

## AC-5. 1차 시장 — 발행 (비동기)

| ID | 시나리오 | 기대 결과 |
|----|----------|-----------|
| AC-5.1 | `requestDeposit` 호출 | 상태 `Pending`, 토큰 미발행 |
| **AC-5.2** | custody attestation 이전에 `claim` 시도 | **revert** (INV-7) |
| AC-5.3 | attestation 후 상태 전이 | `Pending → Claimable` |
| AC-5.4 | `claim` 실행 | 컴플라이언스 검사 후 발행, `Minted` |
| AC-5.5 | 실물 매수 실패 주입 | `Failed` → refund 경로, 대금 반환 |
| AC-5.6 | 환전 지연 주입 (T+2 초과) | `Pending` 유지, 타임아웃 정책대로 처리 |
| AC-5.7 | 모든 상태 전이가 이벤트로 관측됨 | INV-8 |

## AC-6. 1차 시장 — 환매

| ID | 시나리오 | 기대 결과 |
|----|----------|-----------|
| AC-6.1 | `requestRedeem` 호출 | 토큰 잠금, `Pending` |
| AC-6.2 | 잠긴 토큰의 2차 매도 시도 | revert (락업 모듈) |
| AC-6.3 | 실물 매도 완료 후 `fulfillRedeem` | 소각, `totalSupply` 감소 |
| AC-6.4 | 소각 시 적격성 검사 미적용 | 빠져나가는 경로 |
| AC-6.5 | `withdraw` 실행 | 대금 반환, `Settled` |
| AC-6.6 | 환매 후 외국인 비율 하락이 반영됨 | 오라클 갱신 확인 |

## AC-7. 크로스체인 DvP (1차 결제)

| ID | 시나리오 | 기대 결과 |
|----|----------|-----------|
| AC-7.1 | 정상 흐름 | `LOCK → VERIFY → FUND → COMMIT`, 양쪽 완결 |
| **AC-7.2** | 대금 미확보 상태로 deadline 경과 | **ABORT**, 양쪽 에스크로 해제, 자산 원위치 |
| AC-7.3 | 자산 leg만 성공한 상태가 관측되지 않음 | 원자성 확인 |
| AC-7.4 | Finality 정책 미충족 시 commit 보류 | 정책대로 |

## AC-8. 배당

| ID | 시나리오 | 기대 결과 |
|----|----------|-----------|
| AC-8.1 | 배당 선언 후 보유자별 배분 | 투자자별로 개별 계산 |

| AC-8.3 | 세율이 하드코딩돼 있지 않음 | 클레임 기반 조회 (코드 검사) |
| AC-8.4 | 배당이 주가(NAV)에 섞이지 않음 | 별도 분배 |

## AC-9. AI 에이전트

| ID | 시나리오 | 기대 결과 |
|----|----------|-----------|
| AC-9.1 | "하이닉스 500만 원어치 사줘" | 종목·방향·금액 파싱, 종목코드 정규화 후 실행 |
| AC-9.2 | "적당히 좀 사줘" (모호) | `needs_clarification` — 추측하지 않고 되묻기 |

| AC-9.4 | 컨트랙트가 거절한 주문 | `rejected` + 사유 설명. 성공으로 보고하지 않음 |
| AC-9.5 | 한도 초과 거절 후 주문 쪼개기 시도 | **그런 로직이 존재하지 않음** (코드 검사) |
| AC-9.6 | 에이전트가 규제 적격성을 자체 판정하지 않음 | 컨트랙트 결과 수용만 (코드 검사) |

## AC-10. 통합 — 한 바퀴

| ID | 시나리오 |

| **AC-10.1** | 온보딩 → 2차 매수 → 배당 수령 → 2차 매도(Bob) → 1차 환매 → 대금 정산까지 전 구간 통과 |

## 실패 경로 테스트 원칙

> **거절되는 것이 정상 동작이다.** 성공 경로만 테스트한 구현은 미완성으로 간주한다.

각 컴플라이언스 모듈은 최소 1개의 "이 모듈에서 막히는" 테스트를 가져야 하고,
그 테스트는 **어느 모듈에서 막혔는지 구분**할 수 있어야 한다.
